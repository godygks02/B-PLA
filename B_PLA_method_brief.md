# B-PLA Method Brief for Prior-Art and Novelty Search

## 1. Working Title

**B-PLA: Bit-Prefix Piecewise Linear Approximation for Multiplierless Neural Network Inference**

This document summarizes the current B-PLA method so that a literature-search or paper-analysis agent can identify related prior work and evaluate whether the proposed architecture has a defensible novelty position.

## 2. One-Sentence Summary

B-PLA replaces selected floating-point multiplication and nonlinear activation operations in neural network inference with bit-prefix-routed affine approximations that can be evaluated using lookup tables, bit shifts, fixed-point adders, muxes, and packing logic instead of runtime hardware multipliers.

## 3. Core Research Motivation

In neural network inference, a large part of arithmetic energy and area is spent on multiply-heavy datapaths, especially in matrix multiplication, convolution, and activation-related nonlinear function units. Conventional approaches usually optimize the multiplier itself, quantize the model, or approximate activation functions using separate dedicated hardware.

B-PLA starts from a different question:

Can multiplication and activation be treated as two instances of the same hardware-friendly operation pattern?

The intended operation pattern is:

```text
input bits -> prefix routing -> coefficient lookup -> dyadic affine evaluation -> output packing
```

The method is training-free at the model level: trained weights and activations do not need to be retrained. Offline calibration may be used to generate coefficient tables, choose prefix lengths, and quantize coefficients into dyadic or signed power-of-two forms.

## 4. Architecture Overview

B-PLA is a framework with two main arithmetic primitives:

1. **B-PLA Multiplier**
   - Approximates floating-point multiplication.
   - Decomposes FP32 inputs into sign, exponent, and mantissa fields.
   - Computes sign and exponent using low-cost logic.
   - Approximates only the nonlinear mantissa interaction term using 2D prefix-routed affine tiles.
   - Reconstructs and repacks the floating-point result.

2. **B-PLA Activation**
   - Approximates nonlinear activation functions such as ReLU, GELU, sigmoid, tanh, or quick-GELU.
   - Clips or saturates the input to a calibration range.
   - Routes the input by directly extracting FP32 sign, exponent, and mantissa-prefix bits.
   - Selects a 1D affine segment from a coefficient table.
   - Evaluates the local affine approximation using a multiplierless dyadic shift-add datapath.

The intended hardware novelty is not only the individual approximation of multiplication or activation. The stronger architectural idea is that both operators can share a common prefix-routed affine evaluator.

## 5. B-PLA Multiplier Method

### 5.1 FP32 Decomposition

For a normal FP32 input:

```text
x = (-1)^S * 2^E * (1 + M)
```

where:

- `S` is the sign bit.
- `E` is the unbiased exponent.
- `M` is the fractional mantissa in `[0, 1)`.

For two inputs:

```text
x1 = (-1)^S1 * 2^E1 * (1 + M1)
x2 = (-1)^S2 * 2^E2 * (1 + M2)
```

The exact product is:

```text
x1*x2 = (-1)^(S1 xor S2) * 2^(E1 + E2) * (1 + M1 + M2 + M1*M2)
```

The sign and exponent paths are cheap:

```text
Sout = S1 xor S2
Eout = E1 + E2
```

The expensive part is the mantissa interaction term:

```text
M1*M2
```

### 5.2 Mantissa Interaction Approximation

B-PLA approximates only `M1*M2`:

```text
M1*M2 ~= a_ij*M1 + b_ij*M2 + c_ij
```

The indices `(i, j)` are generated by taking the high-order prefix bits of `M1` and `M2`:

```text
i = prefix_k(M1)
j = prefix_k(M2)
```

For FP32, the mantissa field has 23 stored fraction bits. With `k` prefix bits:

```text
i = fraction1_q23 >> (23 - k)
j = fraction2_q23 >> (23 - k)
```

The coefficient table is therefore a compact 2D LUT with:

```text
2^k * 2^k entries
```

Each entry stores affine coefficients:

```text
(a_ij, b_ij, c_ij)
```

### 5.3 Multiplierless Evaluation

To remove runtime multipliers, the coefficients should be constrained to dyadic values or short signed power-of-two expansions:

```text
a ~= sum_t s_t * 2^(-q_t)
b ~= sum_t r_t * 2^(-p_t)
c ~= fixed-point offset
```

Then:

```text
a*M1
```

can be implemented as a small number of shifted versions of `M1` followed by fixed-point additions. The same applies to `b*M2`.

The mantissa path becomes:

```text
approx_cross = shift_add(a_ij, M1) + shift_add(b_ij, M2) + c_ij
mantissa_out = 1 + M1 + M2 + approx_cross
```

This is a fixed-point mantissa-domain computation, not floating-point addition. Floating-point adders are not required inside the mantissa approximation core.

### 5.4 Result Reconstruction

After the approximate mantissa is computed:

```text
mantissa_out = 1 + M1 + M2 + approx(M1*M2)
```

the hardware performs normalization if needed:

```text
if mantissa_out >= 2:
    mantissa_out = mantissa_out / 2
    Eout = Eout + 1
```

Then rounding and FP packing produce:

```text
y = (-1)^Sout * 2^Eout * mantissa_out
```

Special cases such as zero, subnormal, infinity, and NaN can either fall back to conventional handling or be handled by lightweight control logic around the B-PLA core.

## 6. B-PLA Activation Method

### 6.1 Activation Target

B-PLA activation approximates a nonlinear function:

```text
y = f(x)
```

where `f` may be:

- ReLU
- GELU
- quick-GELU
- sigmoid
- tanh
- other inference-time nonlinear functions

The approximation is local affine:

```text
f(x) ~= alpha_i*x + beta_i
```

where index `i` is selected by bit-prefix routing.

### 6.2 FP32 Bit-Field Prefix Routing

The current B-PLA activation frontend avoids expensive runtime range normalization such as:

```text
(x - x_min) / (x_max - x_min)
```

Instead, it uses direct FP32 bit-field extraction. For an FP32 input:

```text
x = (-1)^S * 2^E * (1 + M)
```

The routing address is generated from:

```text
i = addr(S, clamp(E), prefix_k(M))
```

For FP32:

```text
mantissa_prefix = fraction_q23 >> (23 - k)
```

Very small values, zero, or values below a configurable exponent threshold can be routed to a central segment. Values outside the calibrated activation range are clipped or saturated before evaluation.

This routing approach is inspired by the earlier S-PLA bit-extraction idea, but the method is framed as B-PLA activation: a bit-prefix-routed affine approximation module for activation functions.

### 6.3 Coefficient Table

For each route index `i`, a coefficient table stores:

```text
(alpha_i, beta_i)
```

The coefficients can be generated offline by least-squares fitting over representative input samples or over a chosen calibration range.

For true multiplierless hardware, `alpha_i` should be dyadic or a short signed power-of-two expansion:

```text
alpha_i ~= sum_t s_t * 2^(-q_t)
```

Then:

```text
alpha_i*x
```

can be evaluated by bit shifts and fixed-point additions rather than a multiplier.

### 6.4 Activation Datapath

The intended activation hardware flow is:

```text
FP32 x
-> optional clip/saturation
-> sign/exponent/mantissa-prefix extraction
-> route index generation
-> coefficient LUT read
-> dyadic shift-add affine evaluation
-> output normalization/packing if FP output is required
```

If the downstream network uses fixed-point or block-floating formats, the final packing stage can be adapted accordingly.

## 7. Unified Hardware View

The multiplier and activation modules can be implemented separately, but B-PLA's architectural thesis is that they share a common evaluator style:

```text
prefix-routed coefficient lookup + dyadic affine evaluation
```

### 7.1 Multiplier Path

```text
FP x1, FP x2
-> FP decomposition
-> sign XOR and exponent addition
-> 2D mantissa prefix routing
-> coefficient LUT
-> shift-add approximation of M1*M2
-> mantissa reconstruction
-> normalization and packing
```

### 7.2 Activation Path

```text
FP x
-> FP bit-field extraction
-> 1D sign/exponent/mantissa-prefix routing
-> coefficient LUT
-> shift-add approximation of f(x)
-> output packing
```

### 7.3 Possible Hardware Blocks

The B-PLA architecture may contain:

- FP field extractor
- prefix address generator
- coefficient ROM or LUT
- mux/control logic
- barrel shifters or fixed shift wires
- fixed-point adder tree
- mantissa reconstruction adder
- normalization shifter
- rounding logic
- FP or fixed-point packing logic

The B-PLA core is intended to avoid:

- FP mantissa multipliers
- DSP multiplier blocks
- general floating-point adders inside the approximation core
- polynomial evaluation units requiring runtime multipliers

## 8. Expected Benefits

### 8.1 Removal of Runtime Multipliers

The main expected benefit is eliminating hardware multipliers from selected arithmetic paths. In B-PLA multiplication, the exact mantissa product is replaced by a prefix-routed affine approximation. In B-PLA activation, the local affine slope multiplication is replaced by dyadic shift-add evaluation.

### 8.2 Lower Energy Per Operation

A floating-point multiplier is typically much more expensive than bit slicing, LUT access, shifting, muxing, and fixed-point addition. If coefficient tables are compact and coefficients are dyadic, the B-PLA datapath should reduce arithmetic energy.

The energy benefit depends on:

- prefix length
- number of dyadic terms per coefficient
- mantissa precision
- LUT read energy
- fixed-point adder width
- packing and normalization overhead
- whether memory access is included in the accounting

An operation-level model should separately report arithmetic energy and memory/LUT energy.

### 8.3 Hardware Area Efficiency

Replacing multipliers with shifters, muxes, small adders, and coefficient ROMs may reduce area, especially when many multiplier instances are required. The advantage is expected to be stronger when:

- one B-PLA evaluator is reused across multiple operations,
- coefficient tables are small,
- coefficients use one or two signed power-of-two terms,
- activation and multiplication share hardware blocks,
- the design avoids DSP blocks on FPGA or large multiplier arrays on ASIC.

### 8.4 Accuracy Preservation

B-PLA is approximate. Accuracy preservation depends on:

- prefix length,
- dyadic coefficient precision,
- calibration distribution,
- target model sensitivity,
- activation type,
- whether all operations or only selected operations are replaced.

The method should be evaluated by both standalone arithmetic error and end-to-end inference accuracy.

## 9. Distinction from Related Areas

A literature search should compare B-PLA against at least the following areas:

### 9.1 Approximate Multipliers

Relevant prior work may include:

- piecewise-linear approximate multipliers,
- logarithmic multipliers,
- Mitchell multiplier variants,
- approximate mantissa multipliers,
- segmented or table-based multipliers,
- PAM-like or ApproxLP-like designs,
- neural-network approximate arithmetic units.

B-PLA overlaps with this area because the multiplier primitive approximates the mantissa interaction surface with piecewise affine tiles.

The possible distinction is that B-PLA is not only an approximate multiplier. It attempts to use the same prefix-routed affine primitive for both multiplication and nonlinear activation.

### 9.2 Piecewise Linear Activation Units

Relevant prior work may include:

- PWL sigmoid/tanh/GELU approximations,
- LUT-based activation accelerators,
- multiplierless activation functions,
- segment-based activation hardware,
- FPGA activation approximation units.

B-PLA overlaps with this area because activation is approximated by local affine segments.

The possible distinction is the FP32 bit-field prefix routing and the intended unification with the multiplier primitive.

### 9.3 Quantization and Multiplierless Neural Networks

Relevant prior work may include:

- power-of-two quantization,
- shift neural networks,
- multiplierless inference,
- LUT-NN methods,
- product quantization,
- binary/ternary neural networks,
- approximate computing for DNN accelerators.

B-PLA differs from full model quantization because it aims to replace arithmetic operators without necessarily retraining or changing all weights into low-bit integer formats.

### 9.4 ANN-to-SNN and Spike-Based Encoding

The activation routing idea has ancestry in a previous S-PLA direction where FP32 mantissa bits were treated as a deterministic bit-slice spike sequence. For B-PLA, this is not the main method name. The useful part carried forward is direct FP32 bit extraction for routing.

A novelty search should check whether similar bit-field routing has been used for activation approximation, multiplier approximation, or unified arithmetic accelerators.

## 10. Novelty Hypothesis

B-PLA should not claim novelty only as:

```text
piecewise-linear approximation of multiplication
```

or:

```text
piecewise-linear approximation of activation functions
```

Both are known research directions.

The stronger novelty hypothesis is:

```text
B-PLA is a training-free, multiplierless, bit-prefix-routed affine arithmetic framework that treats FP multiplication and nonlinear activation as two instances of one hardware-friendly primitive: coefficient lookup plus dyadic shift-add affine evaluation.
```

Potential novel components to verify through prior-art search:

1. FP32 mantissa-prefix routing for approximate multiplication combined with dyadic affine evaluation.
2. FP32 sign/exponent/mantissa-prefix routing for activation approximation.
3. A unified architecture where multiplier and activation share a prefix-routed affine evaluator.
4. Training-free replacement of selected ANN arithmetic paths using this shared primitive.
5. Energy/area framing where multipliers are removed at the source rather than merely approximated.

## 11. Claims That Need Experimental Support

The following claims should not be stated as final conclusions until supported by experiments or RTL synthesis:

- B-PLA always reduces energy compared with FP multiplication.
- B-PLA always reduces area compared with approximate multipliers.
- B-PLA maintains accuracy for all neural networks.
- B-PLA activation is fully multiplierless unless slopes are dyadic-quantized.
- Python simulation proves hardware multiplier removal.

The current software prototype demonstrates method behavior, routing, and approximation error. A hardware-faithful claim requires:

- dyadic coefficient quantization,
- fixed-point datapath modeling,
- operation-level energy accounting,
- RTL or HLS implementation,
- synthesis proof that multiplier or DSP blocks are not used in the B-PLA core.

## 13. Current Software Modules

The current codebase contains:

- `modules/bpla_multiplier.py`
  - FP32 decomposition.
  - 2D mantissa-prefix routing.
  - affine approximation of `M1*M2`.
  - sign/exponent reconstruction and FP-like output assembly.

- `modules/bpla_activation.py`
  - activation functions such as ReLU, sigmoid, tanh, GELU, and quick-GELU.

  - 1D affine coefficient table fitting.
  - software-level activation approximation.

  - operation-level energy model for the B-PLA multiplier.
  - sweep over prefix length, dyadic term count, and mantissa precision.
  - arithmetic accuracy estimation for hardware-like B-PLA multiplication.

## 14. Short Abstract Draft

B-PLA is a bit-prefix-routed piecewise affine arithmetic framework for energy-efficient neural network inference. Instead of using conventional floating-point multipliers and separate nonlinear activation units, B-PLA decomposes FP32 values into bit fields, uses high-order prefix bits to select local affine coefficient tables, and evaluates the selected approximation with dyadic shift-add logic. For multiplication, B-PLA approximates only the nonlinear mantissa interaction term while preserving simple sign and exponent handling. For activation, B-PLA routes inputs using FP32 sign, exponent, and mantissa-prefix fields and evaluates local affine approximations of nonlinear functions. The intended result is a training-free arithmetic replacement framework that removes runtime multipliers from selected inference paths, improves arithmetic energy efficiency, and may reduce hardware area while preserving model accuracy within a tunable approximation budget.
